# PR#1-2: ID変換ユーティリティ実装

## 概要
netkeiba IDとJV-Link ID間の相互変換機能を実装しました。4つの変換関数を提供し、柔軟な型受け入れとプライベート関数による検証処理の共通化を実現しています。

## 変更内容

### 新規作成ファイル
- `jvlink_db_python/utils/id_converter.py`: ID変換関数の実装
  - `race_id_to_jvlink(race_id, date_id)`: race_idとdate_idをJV-Link形式に変換
  - `date_id_to_race_id(date_id, kaiji, nichiji)`: date_idと開催情報からrace_idを生成
  - `jvlink_to_race_id(year, jyo_cd, kaiji, nichiji, race_num)`: JV-Linkパラメータからrace_idを生成
  - `jvlink_to_date_id(year, month, day, jyo_cd, race_num)`: JV-Linkパラメータからdate_idを生成
  - プライベート関数による検証処理の共通化
    - `_validate_and_convert_year()`: 年の検証と文字列変換
    - `_validate_and_convert_two_digit_number()`: 2桁数値の検証と変換（汎用）
    - `_validate_jyo_cd()`: 競馬場コードの検証
  - 数値パラメータでstr型とint型の両方を受け入れ、ゼロパディング対応
  - None検証の実装
  - race_numの検証範囲を1-12に設定

### 既存ファイルの変更
- `jvlink_db_python/utils/__init__.py`: ID変換関数のエクスポート追加
  - `race_id_to_jvlink`, `date_id_to_race_id`, `jvlink_to_race_id`, `jvlink_to_date_id`をエクスポート
- `mypy.ini`: `warn_unused_ignores`を無効化
  - テストコードの`# type: ignore`が意図的であることに対応
- `doc/PLAN.md`: PR#1-2の実装内容の表記を修正
  - 「IDConverter実装」→「変換関数実装」に変更（クラスではなく関数で実装）
