# PR#0-1: 初期プロジェクト構成

## 概要

jvlink-db-pythonライブラリの基本構造を構築しました。プロジェクトのディレクトリ構成、パッケージ設定、開発環境の設定ファイル、ドキュメント、CI/CDワークフローを整備し、今後の実装のための土台を作成しました。

## 変更内容

### 新規作成ファイル

#### パッケージ設定
- `pyproject.toml`: パッケージメタデータと各種ツール設定
  - パッケージ名、バージョン、依存関係を定義
  - black、isort、mypy、pytestの設定を記載
- `setup.py`: セットアップスクリプト
- `requirements.txt`: 本番環境の依存パッケージ（pandas、PyYAML）
- `requirements-dev.txt`: 開発環境の依存パッケージ（pytest、black、isort、flake8、mypy等）

#### コード品質管理
- `mypy.ini`: mypy静的型チェック設定

#### プロジェクト設定
- `.gitignore`: Git除外設定
  - Pythonの一時ファイル、仮想環境、テストキャッシュ、データベースファイル等を除外
- `.gitmodules`: サブモジュール定義
  - JVLinkToSQLiteとJVLinkToSQLite.wikiを追加

#### 設定ファイル
- `config/config.yml`: デフォルト設定
  - データベースパス、JVLinkToSQLiteパス、取得データ種別の設定

#### ドキュメント
- `README.md`: プロジェクト概要、使用方法、インストール手順
  - 機能紹介、使用例、プロジェクト構成、開発方法を記載

#### CI/CDワークフロー
- `.github/workflows/ci.yml`: GitHub Actions CI設定
  - lint-and-type-check: isort、flake8、mypyによるコード品質チェック
  - test: 単体テスト実行とカバレッジレポート生成
  - integration-test: 結合テスト用（現在はスキップ）

#### Pythonパッケージ構造
- `jvlink_db_python/__init__.py`: メインパッケージ初期化
- `jvlink_db_python/utils/__init__.py`: ユーティリティモジュール初期化
- `jvlink_db_python/models/__init__.py`: データモデルモジュール初期化（将来用）

#### テスト構造
- `test/conftest.py`: pytest全体設定
- `test/integration/conftest.py`: 結合テスト用設定
- `test/unit/manager/conftest.py`: JVLinkDBManager単体テスト用設定
- `test/unit/fetcher/conftest.py`: JVLinkDataFetcher単体テスト用設定
- `test/unit/utils/id_converter/conftest.py`: ID変換ユーティリティ単体テスト用設定
- `test/unit/utils/code_converter/conftest.py`: コード変換ユーティリティ単体テスト用設定
- `test/unit/utils/config/conftest.py`: 設定管理ユーティリティ単体テスト用設定

- `example/race_data_fetch.py`: レースデータ取得例（TODO）
