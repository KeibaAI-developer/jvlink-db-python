# PR#1-1: 例外クラスとユーティリティ基盤

## 概要
jvlink-db-pythonライブラリの基盤となる例外クラスと設定ファイル管理機能を実装しました。全体で使用する基本コードを整備し、今後の機能実装の土台を構築します。

## 変更内容

### 新規作成ファイル
- `jvlink_db_python/exceptions.py`: 例外クラス定義
  - `JVLinkDBError`: 基底例外クラス
  - `DatabaseNotFoundError`: データベース未発見エラー
  - `JVLinkToSQLiteError`: JVLinkToSQLite実行エラー
  - `DataNotFoundError`: データ未発見エラー
  - `IDConversionError`: ID変換エラー

- `jvlink_db_python/utils/config.py`: 設定ファイル管理機能
  - `load_config()`: YAML形式の設定ファイル読み込み
  - `_get_default_config()`: デフォルト設定値の提供
  - `_merge_config()`: ユーザー設定とデフォルト設定の再帰的マージ

### 既存ファイルの変更
- `config/config.yml`: デフォルト設定ファイルの大幅拡充
  - データベースパスとバックアップディレクトリ設定追加
  - JVLinkToSQLite設定の詳細化（実行ファイルパス、setting.xml、throttle_size、log_level）
  - 全データ種別の網羅（蓄積系28種、速報系15種）とカテゴリ分類
  - 各データ種別に対応テーブル名と詳細説明を追記
  - 親子テーブル関係の文書化（例：H1 → NL_H1_HYOSU_ZENKAKE + NL_H1_Hyo*系列）
  - 自動更新設定と更新間隔の設定追加

- `doc/SPEC.md`: 機能仕様書の設定セクション更新
  - config.ymlの構造をSPEC.mdに反映
  - 速報系データ種別の追加（WH, WE, AV, JC, TC, CC, DM, TM）
  - 主要テーブル一覧に速報系テーブルを追加

- `doc/PLAN.md`: 実装計画書の明確化
  - 設定ファイルのキー名を明記（jvlinktosqlite.path, jvlinktosqlite.setting_xml, data_specs.default, update.realtime）
  - レビューポイントに「設定ファイルの構造が実際のconfig.ymlと一致しているか」を追加

- `README.md`: リポジトリURLの修正
  - IssuesリンクのリポジトリURLを`KeibaAI`から`KeibaAI-developer`に変更
